1 Claude Codeをインストール
STEP 1-1
Node.jsを入れる
Claude Codeを動かすために必要な土台です。以下をターミナルに貼り付けて実行してください。
brew install node
▶ このコマンドで何が起きるか
解説
brew = Mac用のアプリインストーラー（Homebrew）
install node = Node.jsというソフトをインストールする

Node.jsは「プログラムを動かすためのエンジン」です。Claude Codeはこのエンジンの上で動くので、先にNode.jsを入れる必要があります。インストール完了まで数分かかる場合があります。
ターミナルの開き方
Macで「command + スペース」→「ターミナル」と入力 → Enterで開きます
STEP 1-2
Homebrewが入っていない場合
上のコマンドで「brew: command not found」と出たら、先にHomebrewを入れてください。
/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"
▶ このコマンドで何が起きるか
解説
Homebrewは「Macのアプリストアのコマンド版」のようなものです。このコマンドはインターネットからHomebrewのインストーラーをダウンロードして実行します。途中でMacのパスワードを聞かれる場合があるので、入力してEnterを押してください（入力中は画面に何も表示されませんが、正常です）。
完了したら、もう一度 brew install node を実行してください。
STEP 1-3
Claude Codeをインストール
npm install -g @anthropic-ai/claude-code
▶ このコマンドで何が起きるか
解説
npm install -g = Node.jsのパッケージ管理ツールで、Macのどこからでも使えるようにインストール（-gは「グローバル」の意味）
@anthropic-ai/claude-code = Anthropic社が公式に提供しているClaude Codeのパッケージ

これが完了すると、ターミナルで claude と打つだけでClaude Codeが起動するようになります。
2 ターミナル環境：cmuxをインストール
Claude Codeは標準のターミナルでも動きますが、cmux（シーマックス）というAI開発専用のターミナルアプリを使うと格段に快適になります。私も普段はcmuxからClaude Codeを使っています。
cmuxの特徴
なぜcmuxを使うのか
  • 1通知リング — Claude Codeが返事を待っている時、タブが青く光ってお知らせしてくれる。複数のClaude Codeを同時に動かしても、どれが注意を必要としているか一目でわかる
  • 2縦タブ + サイドバー — 左側にタブが並ぶので、複数プロジェクトを切り替えやすい。各タブにフォルダ名が自動表示される
  • 3内蔵ブラウザ — Cmd+Shift+L で簡易ブラウザが開く。HTMLのプレビューに便利
  • 4軽量・高速 — macOSネイティブアプリなのでサクサク動く。無料で使える
STEP 2-1
cmuxをインストール
以下の2つのコマンドを順番に実行してください。
brew tap manaflow-ai/cmux
1つ目が完了したら、次を実行：
brew install --cask cmux
▶ このコマンドで何が起きるか
解説
brew tap = cmuxの配布元をHomebrewに登録する（「このお店も使いますよ」と教える感じ）
brew install --cask = Macアプリとしてインストールする

完了すると、アプリケーションフォルダにcmuxが追加されます。
STEP 2-2
cmuxを起動して確認
アプリケーションフォルダからcmuxを開くか、Spotlight（Cmd+スペース）で「cmux」と入力して起動してください。
確認ポイント：
  • ✓Cmd+T で新しいタブが開き、左サイドバーにフォルダ名が表示される
  • ✓claude と入力してEnterを押すと、Claude Codeが起動し、タブに青いリングが表示される
  • ✓Cmd+Shift+L で内蔵ブラウザが開く
STEP 2-3
テーマを設定する（任意）
cmuxはGhosttyベースなので、見た目のカスタマイズができます。ライトテーマにしたい場合は、以下のファイルを作成してください。
mkdir -p ~/.config/ghostty && cat > ~/.config/ghostty/config << 'EOF' # テーマ background = f5f5f5 foreground = 2e3440 # 半透明（デスクトップがうっすら透ける） background-opacity = 0.92 # フォント font-size = 14 # ウィンドウ余白 window-padding-x = 12 window-padding-y = 8 window-theme = light EOF
▶ このコマンドで何が起きるか
解説
cmuxの見た目を設定するファイルを作成しています。明るい背景色（#f5f5f5）に暗い文字色（#2e3440）のライトテーマです。

background-opacity = 0.92 にすると、デスクトップの壁紙がうっすら透けて見えるおしゃれな見た目になります。

設定を変えたら Cmd+Q でcmuxを終了し、もう一度起動すると反映されます。
テーマ設定は後からでもOK
まずはデフォルトのまま使い始めて、慣れてきたら好みに合わせてカスタマイズしてください。
以降のコマンドはcmuxで実行
ここからの手順は、標準のターミナルではなくcmuxを開いて実行してください。cmuxの方が作業が快適です。
3 APIキーを設定
STEP 4-1
Anthropic ConsoleでAPIキーを取得
ブラウザで Anthropic Console にアクセスし、APIキーを発行してください。
APIキーは sk-ant-... で始まる文字列です。
STEP 4-2
環境変数に設定する
重要: Claude Codeではなくターミナルで実行してください
このコマンドにはAPIキーが含まれます。Claude Codeのチャットには絶対に貼らないでください。
必ず「ターミナル」アプリを別で開いて、そこに貼って実行してください。
（Claude Codeのチャットに貼ると、キーがAnthropicのサーバーに送信されてしまいます）
「ここにAPIキーを貼る」の部分を実際のキーに置き換えて、ターミナルに貼り付けてください。
echo 'export ANTHROPIC_API_KEY="ここにAPIキーを貼る"' >> ~/.zshrc && source ~/.zshrc
▶ このコマンドで何が起きるか
解説
echo '...' >> ~/.zshrc = Macの設定ファイル（.zshrc）にAPIキーの設定を1行追記します。.zshrcはターミナルを開くたびに自動で読み込まれるファイルです。
source ~/.zshrc = 追記した設定を今すぐ反映します（通常は次にターミナルを開いた時に反映されますが、これで即座に使えるようになります）。

つまり「APIキーをMacに記憶させて、Claude Codeが自動でそのキーを使えるようにする」コマンドです。一度実行すれば、以降は毎回キーを入力する必要はありません。
注意
APIキーは絶対に他の人に見せない・チャットに貼らないでください。
Claude Codeとの会話の中にもキーを貼らないでください。
4 作業フォルダを作る
STEP 4-1
Claude Code用のフォルダを「書類」の中に作る
Claude Codeは「今いるフォルダ」を作業場所として認識します。なので、まずClaude Code用の作業フォルダを1つ作っておきます。
mkdir -p ~/Documents/Claude_Work
▶ このコマンドで何が起きるか
解説
mkdir -p = フォルダを作るコマンド（-pは「途中のフォルダも含めて作る」の意味）
~/Documents/Claude_Work = Finderで言うと「書類」フォルダの中に「Claude_Work」というフォルダを作る

このフォルダがClaude Codeの「ホームグラウンド」になります。今後、プロジェクトごとにこの中にサブフォルダを作っていきます。

Finderで見ると：書類 → Claude_Work という場所です。
STEP 4-2
このフォルダにCLAUDE.mdを作る
プロジェクトフォルダ専用のルールファイルを配置します。Claude Codeはこのフォルダで起動した時にこのファイルを読み込みます。
cp ~/claude-starter-kit/.claude/CLAUDE.md ~/Documents/Claude_Work/CLAUDE.md
▶ なぜ2箇所にCLAUDE.mdがあるのか
解説
Claude Codeの設定ファイルには2つのレベルがあります：

1. ~/.claude/CLAUDE.md（グローバル設定）
どのフォルダで起動しても常に読み込まれる。全プロジェクト共通のルール。
→ STEP 4でコピーするスターターキットに含まれています。

2. プロジェクトフォルダ内のCLAUDE.md（プロジェクト設定）
そのフォルダで起動した時だけ読み込まれる。プロジェクト固有のルール。
→ このSTEPで配置するファイルです。

Claude Codeは両方を読み込んで合算するので、グローバルの安全ルール＋プロジェクト固有のルールの両方が適用されます。

今後、SNS自動運用プロジェクト固有のルールが増えたら、このプロジェクトフォルダ内のCLAUDE.mdに追記していきます。
フォルダの使い方
今後Claude Codeを使う時は、このフォルダに移動してから起動します：
cd ~/Documents/Claude_Work && claude
5 スターターキットを配置
STEP 4-1
このフォルダの .claude をホームにコピー
このHTMLと同じフォルダにある .claude フォルダを、自分のホームフォルダにコピーします。
cp -R ~/claude-starter-kit/.claude ~/
▶ このコマンドで何が起きるか
解説
cp -R = フォルダごとまるっとコピーするコマンド
~/claude-starter-kit/.claude = コピー元（このスターターキットの中の設定フォルダ）
~/ = コピー先（あなたのMacのホームフォルダ）

つまり、スターターキットの設定ファイルが、あなたのMacの「~/.claude/」にコピーされます。

Claude Codeは起動するたびに「~/.claude/」フォルダを自動的に読み込みます。なので、ここに設定ファイルを置くだけで以下が全て有効になります：

・スキル17個（YouTube文字起こし、X検索、深掘り調査など）
・安全ルール（ファイル削除ブロック、機密情報保護など）
・セキュリティフック（危険な操作を自動で止める仕組み）
・作業の自動引き継ぎ（セッションが切れても前回の続きから再開）
・デザインの好み設定（HTML生成時のテーマカラーやスタイル）

特別な設定作業は不要で、コピーするだけでClaude Codeが最初から高機能な状態で使えます。
補足
もしフォルダの場所が違う場合は、このスターターキットのフォルダがある場所に合わせてパスを変えてください。
例: USBで渡された場合 → cp -R /Volumes/USBの名前/claude-starter-kit/.claude ~/
STEP 4-2
hookスクリプトに実行権限を付ける
セキュリティ機能を動かすために必要です。
chmod +x ~/.claude/hooks/*.sh
▶ このコマンドで何が起きるか
解説
chmod +x = ファイルに「実行してもいいよ」という許可を与えるコマンド
~/.claude/hooks/*.sh = .claude/hooksフォルダ内の全てのスクリプトファイル

Macではセキュリティ上、ファイルをコピーしただけでは「実行権限」が付きません。このコマンドでセキュリティフック（危険な操作を自動でブロックするスクリプト）に実行権限を付けます。

これを忘れると、安全インストーラーやセッション切れ前の自動保存が動きません。

C CLAUDE.md -- 全ルール解説
CLAUDE.mdは「Claude Codeの振る舞いを決めるルールブック」です。このファイルがあることで、Claude Codeが勝手に危険なことをしたり、分かりにくい説明をしたりすることを防いでいます。
以下、各ルールの意味と「なぜこのルールがあるのか」を解説します。
セキュリティ 機密情報の予防・検知
APIキーやパスワードをClaude Codeに渡さない・見せないルール。スクリーンショットに機密情報が映っていた場合も即座に警告する。
▶ なぜこのルールがあるか
背景
Claude Codeとの会話内容は、AIの処理のためにAnthropicのサーバーに送信されます。つまり、会話の中にAPIキーを貼り付けると、そのキーがAnthropicのサーバーを経由することになります。万が一の情報漏洩リスクを考えると、機密情報は「Claude Codeには絶対に渡さない」のが最も安全です。

このルールにより、Claude Code側から「キーは私に見せず、別のターミナルで.envファイルに直接貼ってください」と案内してくれます。
対応ファイル: CLAUDE.md 安全ルール §5
セキュリティ .claudeignore（プロジェクト作成時に必須）
新しいプロジェクトを作る時、最初に.claudeignoreファイルを設置する。これにより.envなどの機密ファイルがClaude Codeから見えなくなる。
▶ なぜこのルールがあるか
背景
.claudeignoreがない状態だと、Claude Codeはプロジェクト内の全ファイルを読めてしまいます。.envファイルにはAPIキーやデータベースのパスワードが書いてあるので、Claude Codeがそれを読み取って会話に含めてしまうと、意図せず機密情報がAnthropicサーバーに送られるリスクがあります。

.claudeignoreを設置することで、「そもそもClaude Codeには見えない」状態を作れます。
対応ファイル: CLAUDE.md 安全ルール §4
セキュリティ printenv / env コマンド実行禁止
環境変数を一覧表示するコマンド（printenv, env, set等）は絶対に実行しない。
▶ なぜこのルールがあるか
背景
.claudeignoreで.envファイルを隠しても、printenvコマンドを実行するとOSレベルで環境変数が全て表示されます。APIキーは通常環境変数に設定するため、printenvの出力にキーがそのまま含まれます。

つまり、.claudeignoreの「ファイルを見せない」防御を、コマンド経由で迂回できてしまうのです。このルールで、その抜け道を塞いでいます。
対応ファイル: CLAUDE.md 安全ルール §5 予防セクション
セキュリティ スキルの安全インストール
ネットからスキルを追加する時、npx skills add は使わず、専用の安全インストーラー（safe-skill-install.py）を使う。
▶ なぜこのルールがあるか
背景
npx skills add を使うと、インターネットからスキルファイルをダウンロードして直接実行します。もしそのスキルに悪意のあるコード（リバースシェル、データ送信、ファイル削除等）が含まれていた場合、PCが乗っ取られたり情報が盗まれるリスクがあります。

safe-skill-install.pyは、ダウンロード後に「セキュリティ検査」を自動で行います。危険なパターン（外部へのデータ送信、隠されたコード実行等）を検知したら、保存せずにブロックします。hookで npx skills add を自動ブロックしているので、うっかり使おうとしてもClaude Codeが止めてくれます。
対応ファイル: CLAUDE.md スキル安全インストール / scripts/safe-skill-install.py / hooks/guard-remote-skills.sh
ワークフロー 大タスク前の自動セーブ
大きな作業（HTML生成、複数エージェント並行実行等）を始める前に、自動でフィードバックとHANDOVER（引き継ぎ書）を保存する。
▶ なぜこのルールがあるか
背景
Claude Codeにはトークン（会話の長さ）の上限があります。大きな作業はトークンを大量に消費するため、作業途中でセッションが切れることがあります。

セッションが切れると、それまでの会話内容やユーザーが伝えたフィードバックが全て失われます。同じことを何度も説明し直す無駄が発生します。

このルールにより、大きな作業の前に「ここまでで分かっていること」を自動保存するので、万が一セッションが切れても、次のセッションですぐに続きから再開できます。
対応ファイル: CLAUDE.md 大タスク前の自動セーブ
ワークフロー トークン使用量の自己監視
10-20ターンごとにトークン使用量をチェック。75%で引き継ぎ書を自動生成、90%で作業を一時停止する。
▶ なぜこのルールがあるか
背景
Claude Codeのセッションにはトークン上限（約200,000トークン）があり、上限に達するとセッションが突然切れます。何の前触れもなく切れるため、作業途中の内容がすべて失われます。

これを何度か経験した結果、「セッションが切れる前に、自分で気づいて引き継ぎ書を作っておく」ルールが生まれました。75%到達後は毎ターンチェックするので、「気づいたら100%でした」という事態を防げます。
対応ファイル: docs/token-monitoring.md
ワークフロー セッション開始時の自動ハンドオーバー確認
新しいセッションを始めた時、前回の引き継ぎ書（HANDOVER.md）があれば自動で検出し、続きから再開するか確認してくれる。
▶ なぜこのルールがあるか
背景
上のトークン監視で引き継ぎ書を作っても、次のセッションで「その引き継ぎ書を読んで」と手動で伝えるのは面倒です。このルールにより、セッション開始時に自動で前回の引き継ぎ書を検出し、「続きをやりますか？」と聞いてくれます。

「はい」なら前回の続きから、「いいえ」なら引き継ぎ書をアーカイブして新規スタートします。
対応ファイル: docs/session-start.md
セキュリティ 既存ファイルの上書き禁止（確認必須）
既存のファイルを編集・上書きする前に、必ず確認メッセージを出す。可能ならバックアップも作る。
▶ なぜこのルールがあるか
背景
Claude Codeはデフォルトだと、ファイルを確認なしに上書きすることがあります。これは便利な反面、意図しない変更でファイルが壊れるリスクがあります。特に長時間かけて作ったHTMLやコード、設定ファイルが確認なしに上書きされると、元に戻すのが大変です。

このルールで「上書きしていいですか？」という確認が必ず入るようになります。
対応ファイル: CLAUDE.md 安全ルール §1
セキュリティ 削除コマンドの実行禁止
rm、rm -rf、rmdir などの削除系コマンドは原則実行しない。どうしても必要な場合は理由を説明して承認を求める。
▶ なぜこのルールがあるか
背景
rm -rfコマンドは「指定したフォルダ以下を確認なしで全削除する」非常に強力なコマンドです。一度実行すると元に戻せません。AIが判断を間違えて重要なファイルやフォルダを削除してしまうリスクを排除するため、このルールがあります。
対応ファイル: CLAUDE.md 安全ルール §2
セキュリティ パッケージ追加は事前説明と承認が必要
npm install等でパッケージ（追加機能）を入れる前に、何を・なぜ・どの範囲に入れるかを説明し、承認を得てから実行する。
▶ なぜこのルールがあるか
背景

このルールで、Claude Codeが勝手にパッケージを入れることを防ぎ、何が入るか理解した上で判断できるようにしています。
対応ファイル: CLAUDE.md 安全ルール §3

技術的なコマンドを実行する前に、日本語で分かりやすく「これは○○をするコマンドです」と説明し、OKをもらってから実行する。

プラン詳細 1
API従量課金（このガイドのSTEP 3で設定した方式）
APIキーを使ってClaude Codeを動かす方式です。使った分だけ課金されます。

料金の目安:
・軽い作業（質問、調べ物、短い指示）: 1セッションあたり数十円〜数百円
・重い作業（大規模なHTML生成、長い調査）: 1セッションあたり数百円〜千円程度
・1日の通常利用: 数百円〜2,000円程度

モデルによる価格差: Opusモデル（高性能）はSonnet（標準）の約1.7倍高い。軽い作業はSonnetで十分なので、/model で切り替えるとコスト節約になります。
使用額の確認方法
Claude Code内で /cost と打つと、そのセッションでいくら使ったか確認できます。
必須
API従量課金の場合：月額上限を必ず設定する
使いすぎを防ぐために、Anthropic Consoleで月額の上限を設定できます。
  • 1ブラウザで console.anthropic.com にログイン
  • 2Settings（設定）→ Billing（課金）→ Usage Limit（利用上限）に進む
  • 3月額の上限金額を設定する（最初は$50〜$100程度がおすすめ）
注意
上限を設定しないと、使った分だけ際限なく課金されます。最初に必ず上限を設定してから使い始めてください。

Pro（$20/月）: Web版メインでClaude Codeも少し使いたい人向け。ヘビーに使うとすぐ上限に達します。

Max 5x（$100/月）: 毎日Claude Codeを使う人に最適。API従量課金で月$100前後かかっているなら、こちらの方が安心。Opusモデルも使える。

Max 20x（$200/月）: 1日中Claude Codeを回す人向け。使用量がMax 5xで足りない場合に。
使用量の上限について
Pro/Maxプランの使用量は週ごとにリセットされます。上限に達しても翌週にはまた使えます。Web版チャットとClaude Codeで使用量は共有です。
判断ガイド
どのプランをどのタイミングで選ぶか
まずはAPI従量課金で始めて、使用量に応じてプラン変更するのがおすすめです。
ステージ おすすめプラン 判断基準
最初の1〜2週間 API従量課金 まず実際の使用量を知る。/cost で毎日確認
月$50以下だった API従量課金のまま or Pro($20) このままで十分。Proにすると定額で安心感が増す
月$50〜$100 Max 5x ($100) 定額で予算が読める。使い放題に近い安心感
月$100超 Max 20x ($200) ヘビーユーザー。定額の方が確実に安い
要注意
API従量課金 → Pro/Maxに切り替える時の落とし穴
Pro/Maxプランに加入しても、APIキーの設定が残っているとAPI従量課金が止まりません。
つまりサブスク代 + API従量課金の二重課金になるリスクがあります。
なぜこうなるのか
Claude Codeは環境変数 ANTHROPIC_API_KEY が設定されていると、サブスクのアカウントではなくAPIキーを優先して使ってしまう仕組みです。Pro/Maxに入ったのにAPIキーが残っていると、サブスクは使われず、APIの方に課金が走ります。
正しい切り替え手順:
  • 1claude.ai/settings/billing でPro/Maxプランに加入する
  • 2APIキーの設定を無効にする（以下のコマンドを実行）
sed -i '' '/ANTHROPIC_API_KEY/d' ~/.zshrc && source ~/.zshrc
▶ このコマンドで何が起きるか
解説
sed -i '' = ファイルを直接編集するコマンド
'/ANTHROPIC_API_KEY/d' = ANTHROPIC_API_KEYという文字を含む行を削除する
~/.zshrc = Macの設定ファイル（STEP 3-2で書き込んだファイル）
source ~/.zshrc = 変更を即座に反映する

つまり「STEP 3-2で追加したAPIキーの設定行を消す」コマンドです。これでClaude Codeがサブスクのアカウントを使うようになります。
  • 3Claude Codeで /logout → /login を実行して、サブスクのアカウントでログインし直す
  • 4cmuxを一度閉じて再起動する（Cmd+Q → 再度cmuxを開く）
確認方法
切り替え後にClaude Codeで /cost と打って「APIキーが設定されていません」的なメッセージが出ればOK。サブスクのアカウントで動いています。
A 自動承認モード（Dangerously Skip Permissions）
概要
毎回の確認をスキップして自動で実行させる
Claude Codeはデフォルトだと、ファイルの読み書きやコマンドの実行のたびに「実行しますか？」と確認を求めてきます。安全ですが、毎回Yesを押すのが面倒です。

自動承認モードをONにすると、Claude Codeが判断して自動で実行してくれます。このスターターキットには安全ルール（削除ブロック、上書き確認等）が組み込まれているので、自動承認モードでも危険な操作はちゃんとブロックされます。
切り替え方法
Shift + Tab で切り替え
Claude Codeが起動している状態で Shift + Tab を押すと、承認モードが切り替わります。
  • 1通常モード -- 毎回確認される。最初はこれで慣れるのがおすすめ
  • 2自動承認モード -- 確認なしで実行される。慣れてきたらこちらに切り替え
おすすめの流れ
最初の数日は通常モードで使って、「どんな確認が来るか」を体感してください。
慣れてきたらShift + Tabで自動承認モードに切り替えると、作業がサクサク進みます。

Y/N 承認を求められた時の判断ガイド
通常モードで使っていると、Claude Codeが「実行しますか？」と確認してきます。YesかNoか迷った時の判断基準です。
Yes OKを出してよい場合
・ファイルの読み込み（Read）-- 読むだけなので安全です
・ファイルの検索（Grep, Glob）-- 探すだけなので安全です
・新しいファイルの作成（Write）-- 既存ファイルには影響しません
・Web検索やWebページの取得（WebFetch, WebSearch）-- 調べるだけなので安全です
・git commit -- 変更の保存ポイントを作るだけです
確認 内容を確認してからOKする場合
・ファイルの編集（Edit）-- 何をどう変えるか表示されるので、内容を確認してからOK
・Bashコマンドの実行 -- Claude Codeが「○○するコマンドです」と説明してくれるので、納得できたらOK
・パッケージのインストール -- 何を入れるか確認してからOK
No 止めるべき場合
・APIキーや機密情報を含むコマンドの実行
・rm（削除）コマンド -- 安全ルールでブロックされるはずですが、念のため
・意味が分からない操作 -- 「これ何？」と聞き返せばOK。Claude Codeが日本語で説明してくれます
? よくあるトラブルと対処法
Q 英語のエラーメッセージが出た
対処: そのまま「このエラーは何？どうすればいい？」とClaude Codeに聞いてください。日本語で原因と対処法を説明してくれます。自分で読む必要はありません。
Q Claude Codeが反応しない・固まった
対処: Ctrl + C を押してください。処理が中断されます。もう一度Ctrl + Cを押すと終了します。その後、もう一度 cd ~/Documents/Claude_Work && claude で再起動すればOKです。
Q 「API key not found」と言われた
対処: STEP 2のAPIキー設定がうまくいっていません。ターミナルで以下を実行して、キーが設定されているか確認してください（Claude Codeではなくターミナルで）。
echo $ANTHROPIC_API_KEY
何も表示されなければ、STEP 3-2をやり直してください。
Q 「command not found: claude」と言われた
対処: Claude Codeのインストールがうまくいっていません。STEP 1-3の npm install -g @anthropic-ai/claude-code をもう一度実行してください。
Q 「command not found: brew」と言われた
対処: Homebrewが入っていません。STEP 1-2のコマンドを先に実行してください。
Q 思った結果と違うものが出てきた
対処: 「これじゃなくて、○○してほしい」と日本語で伝え直してください。Claude Codeは指示の修正に柔軟に対応します。具体的に「どこが違うか」「どうしてほしいか」を伝えるほど、正確に修正してくれます。
Q 「トークン75%に達しました」と表示された
対処: これはエラーではありません。Claude Codeが自動で引き継ぎ書を作ってくれるので、現在のセッションを終了し、新しいセッション（ claude で再起動）で続きから再開してください。

  • 2/help を打つ -- Claude Codeの基本的な使い方が表示されます
  • 3私に聞く -- 上の2つで解決しない場合は、私に相談してください

  • 2フィードバックを伝える -- 「こうしてほしい」「これは違う」と伝えると、Claude Codeがスキルに学習内容を保存します。次回から同じミスをしなくなります
