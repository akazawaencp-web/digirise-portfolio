裏方A君 開発ガイド

ゼロから作るLINE自動調査AIシステム

LINEに話しかけるだけでタスクを記録し、AIが自動で調査してHTMLレポートを届けてくれる。
そんなシステムを、Claude Codeと一緒にゼロから作るためのガイドです。
CONFIDENTIAL

このガイドの内容は機密情報です。
RIOが構築した独自のシステム設計・スキル・ノウハウを含みます。外部への公開・共有・転載は禁止です。スクリーンショットの撮影やURLの共有もしないでください。

MOST IMPORTANT
このガイドを進める上で一番大事なこと
このプロジェクトを通して、わからないことがあったらどんどんClaude Codeに質問してください。

エラーが出た、画面の意味がわからない、次に何をすればいいかわからない — 全部、Claude Codeに聞けば大丈夫です。

特に有効なのが、スクリーンショットを撮ってClaude Codeに貼り付ける方法です。「この画面のここがわからない」と画像を添えて聞くだけで、的確に教えてくれます。

「自分で調べて解決しよう」という癖をなくしてください。

Google検索で30分悩むよりも、Claude Codeに聞けば10秒で答えが返ってきます。AI時代の新しい仕事の仕方は「自力で頑張る」ではなく「AIに正しく頼る」です。遠慮なく、何度でも聞いてください。
OVERVIEW
裏方A君とは
「超有能な裏方Aくん」は、LINE上で動くAIアシスタントです。

LINEに音声入力やテキストで思いついたことを送ると、それがタスクとして自動登録されます。登録されたタスクはAIが定期的に「やった？」と催促（チェイス）してくれます。さらに、調査が必要なタスクには自動で深掘り調査（DeepDive）を行い、結果をHTMLレポートにしてLINEに届けてくれます。

DEEPDIVE

DASHBOARD

ポイント
サーバー（Railway）はインターネット上で24時間動き続ける部分。LINEからのメッセージを受け取り、定期通知を送る。
Mac側（Claude Code）はMacが起きている間だけ動く部分。1時間ごとに未処理タスクを調査してHTMLレポートを作り、サーバーにアップロードする。

PART 2
サーバーを構築する（Railway + FastAPI）
所要時間: 約90分（最も時間がかかるパート）
ここが一番大変なパートですが、コードは全てClaude Codeが書いてくれます。あなたがやることは「Claude Codeに指示を出す」「エラーが出たらClaude Codeに貼り付ける」だけです。

用語のヒント: 下線付きの単語にカーソルを合わせると説明が表示されます。
1GitHubアカウントを作り、リポジトリを作る
GitHub（ギットハブ）はコードを保存・管理するサービスです。Railwayと連携するために必要です。
https://github.com/signup
  • 1上のURLを開き、メールアドレス・パスワード・ユーザー名を設定してアカウント作成（無料プランでOK）
  • 2ログイン後、画面右上の「+」アイコン → 「New repository」をクリック
  • 3Repository nameに task-chase と入力
  • 4Private（非公開）を選択 — コードを他の人に見られないようにするため
  • 5「Create repository」ボタンをクリック
▶ GitHubとは何か、なぜ必要か
GitHubは「コードの保管庫」のようなものです。あなたのMacで書いたコードをGitHubにアップロード（push）すると、Railwayが自動でそれを読み取ってサーバーに反映してくれます。つまり「Mac → GitHub → Railway」という橋渡しの役割です。

「リポジトリ」は「プロジェクト1個分の保管場所」のことです。裏方A君のコード一式がここに入ります。
2プロジェクトフォルダを作る
cmuxで以下を実行してプロジェクトフォルダを作り、Claude Codeを起動します。
mkdir -p ~/Documents/Claude_Work/task-chase && cd ~/Documents/Claude_Work/task-chase && claude
3Claude Codeにプロジェクトの骨組みを作ってもらう
Claude Codeが起動したら、以下のように指示します。
あなた > LINE Botのサーバーをゼロから作りたいです。 以下の仕様で作ってください： ・Python + FastAPI ・LINE Messaging API v3 でWebhookを受け取る ・LINEに送られたテキストをClaude API（Sonnet）で解析して タスクとしてJSONファイルに保存する ・タスクにはジャンル、タイプ、期限を自動で付ける ・環境変数は.envで管理 ・.claudeignoreと.gitignoreも作って ・Railwayにデプロイする前提で
Claude Codeがやってくれること
この指示だけで、Claude Codeが以下を全て自動で作ります：
・Pythonコード（main.py, task_parser.py 等）
・requirements.txt（必要なライブラリ一覧）
・Procfile（Railwayでの起動設定）
・.env.example（環境変数のテンプレート）
・.claudeignore / .gitignore（セキュリティ設定）
RIOからひとこと
上のプロンプト（指示文）を見て「最初からこんな整理された指示は出せないよ」と思うかもしれません。安心してください。自分もいきなりこんな指示を出したわけではありません。

最初は「LINE Botを作りたい」くらいのざっくりした指示から始めて、Claude Codeと何回もやり取りを重ねながら、少しずつ形にしていきました。上のプロンプトは、その試行錯誤の結果としてまとまったものです。

だから最初から完璧な指示を出そうとしなくていいです。まずは「LINE Botを作りたい」だけでもOK。Claude Codeが「どんな仕様にしますか？」と聞いてくるので、会話の中で仕様が固まっていきます。

ただ、このガイドを通して経験してほしいのは、「問い（指示）の精度を上げる力」の重要性です。同じClaude Codeを使っていても、指示の出し方で結果は大きく変わります。「自分が本当に欲しいものは何か」を言語化する力 — これがAI時代の「ビジネス戦闘力」です。プログラミングの知識ではなく、この「問いの力」を磨くことが、今後ますます重要になります。
4.envファイルを設定する

Claude Codeは閉じる必要はありません。別タブで作業するだけです。
1行目: プロジェクトフォルダに移動する
cd ~/Documents/Claude_Work/task-chase
2行目: 見本ファイルをコピーして.envファイルを作る

3行目: .envファイルを開いて編集する

▶ nanoエディタの使い方
nano はターミナル内で動くシンプルなテキストエディタです。
・矢印キーでカーソルを移動して、値を書き換えます
・書き換えが終わったら Ctrl+O（保存）→ Enter（確定）→ Ctrl+X（閉じる）
・間違えたら Ctrl+X → 「N」で保存せずに閉じてやり直せます
設定する主な値：

PART 3
チェイス機能を追加する（定期催促通知）
所要時間: 約30分
登録されたタスクを定期的に「やった？」と催促する機能を追加します。ストイックなトレーナーキャラで通知が来るので、サボれません。
1Claude Codeにチェイス機能を作ってもらう
Claude Codeに以下のように指示します。
あなた > チェイス（催促通知）機能を追加してください。 ・朝8時、昼12時、夕方18時にLINE pushメッセージを送る ・未完了のタスクからランダムに最大3件を選んで催促する ・キャラクターは「ストイックなパーソナルトレーナー」風 ・催促文はClaude API（Haikuモデル。安いので）で毎回生成 ・APScheduler（時間指定で自動実行する仕組み）を使ってスケジューラーを実装して
2LINEコマンドを追加する
LINEから使えるコマンドも追加してもらいます。
あなた > LINEで以下のコマンドが使えるようにしてください： ・「完了報告」→ 完了するタスクを選んで報告 ・「タスク一覧」→ 現在のタスク一覧を表示 ・「今日のタスク」→ 今日期限のタスクだけ表示 ・「今週のまとめ」→ 週間サマリー ・「費用」→ 今月のAPI使用料を表示 ・「あとでやる」→ タスクを後回しにする ・上記以外のテキスト → 全てタスクとして登録

PART 4
DeepDive機能を構築する（自動調査AI）
所要時間: 約90分
ここが裏方A君の最大の目玉機能です。登録されたタスクの中で「調査が必要なもの」を自動で深掘りし、HTMLレポートにまとめてLINEに届けてくれます。

サーバー側（Railway）とMac側（Claude Code）の連携で動く仕組みです。 
LINEでタスク登録
      |
      v
Railway: タスクDBに保存（deepdive_status: "pending"）
      |
      | 1時間ごとにチェック
      v
Mac (Claude Code ヘッドレスモード)
      |
      | 1. Railwayから未処理タスクを取得
      | 2. Web検索 + YouTube字幕取得で調査
      | 3. HTMLレポートを生成
      | 4. RailwayにHTMLをアップロード
      | 5. LINEに通知を送る
      v
あなたのLINEにレポートURLが届く
1Railway側にDeepDive APIを追加する
Claude Codeに以下を指示します。
あなた > DeepDive用のAPIエンドポイントを追加してください： ・GET /api/deepdive/tasks → 未処理タスク一覧を返す ・POST /api/deepdive/upload → HTMLファイルをアップロード ・POST /api/deepdive/skip → タスクをスキップ ・POST /api/deepdive/notify → LINE通知を送る ・POST /api/deepdive/check-request → 手動リクエスト確認 ・全てBearer認証（合言葉のような認証方式。DEEPDIVE_API_KEYで保護）で保護 ・HTMLは /reports/ パスで配信する ・Railway Volume（サーバー再起動してもデータが消えない保存領域）を /data にマウントして永続化
2deepdiveスキルを作る
RIOからスキルを受け取っている場合
RIOからdeepdiveスキル（SKILL.md）を渡されている場合は、このステップは不要です。渡されたファイルを ~/.claude/skills/deepdive/ に配置するだけでOKです。
以下はゼロから自分で作る場合の手順です。
Mac側で動くClaude Codeの「スキル」を作ります。スキルとは、Claude Codeに特定の手順を教え込む仕組みです。
あなた > deepdiveスキルを作りたいです。以下の手順で動くスキルです： 1. scripts/deepdive_client.py fetch で未処理タスク取得 2. タスクのraw_inputを解析（URLがあればアクセスして内容取得） 3. WebSearchで最低3回検索して関連情報を集める 4. YouTube動画があれば字幕を取得 5. 調査結果をHTMLレポートにまとめる 6. scripts/deepdive_client.py upload でRailwayにアップロード 7. scripts/deepdive_client.py notify でLINE通知 SKILL.mdをスキルフォルダに作成してください。
スキルの保存場所
~/.claude/skills/deepdive/SKILL.md に保存されます。
このファイルがあると、Claude Codeに「深掘りして」と言うだけでスキルが起動します。
3deepdive_client.pyを作る
Railway APIとやり取りするためのスクリプトです。Claude Codeに作ってもらいます。

▶ ヘッドレスモードとは
通常のClaude Codeは対話型（あなたが打って、AIが返す）ですが、ヘッドレスモードは指示を渡すだけで自動実行させるモードです。

claude -p "deepdiveスキルを実行して" --model sonnet

これで人が操作しなくてもClaude Codeが自動で調査→レポート生成→アップロードまでやってくれます。
5LINEから手動で深掘りを依頼できるようにする
自動実行とは別に、LINEで「深掘り」と送ったら即座に調査を開始するようにします。
あなた > LINEで「深掘り」と送った時に、最新の未処理タスクの deepdiveを即座に開始する機能を追加してください。 ・Railway側にリクエストフラグを立てるAPI追加 ・Mac側にdeepdive-watcher.sh（5分ごとにフラグ確認）を作成 ・フラグが立っていたらauto-deepdive.shを起動
重要: Macのスリープ防止について
自動実行（auto-deepdive.sh / deepdive-watcher.sh）はMacが起きていて、Wi-Fiに繋がっている時だけ動きます。
Macがスリープすると自動実行は止まってしまいます。

対策: Amphetamine（アンフェタミン）というMac用の無料アプリを入れます。
このアプリを使うと、Macを「寝かさない」状態に保てます。
6Macのスリープを防止する（Amphetamine）
インストール:
Mac App Storeで「Amphetamine」と検索 → 無料でインストール

設定:
1. インストールするとメニューバー（画面上部の右側）に錠剤型のアイコンが出る
2. アイコンをクリック → 「Indefinitely」（無期限）を選ぶ
3. これだけでMacがスリープしなくなる

なぜ必要？
auto-deepdive.shは1時間ごと、deepdive-watcher.shは5分ごとに動いています。Macがスリープするとこれらが全部止まるため、タスクを登録しても深掘り調査が実行されません。
Amphetamineを有効にしておけば、外出中でもMacが起きたままなので、自動調査が止まりません。

注意点:
・電源ケーブルは繋いでおくこと（バッテリーだけだと減りが早い）
・Wi-Fiが切れると通信できないので、安定したネット環境で
・外出時や寝る時も動かし続けたい場合はAmphetamineをオンにしたまま

▶ RIOのダッシュボードはこうやって作った
RIOが実際にやったデザインの進め方を参考に紹介します。

Step 1: まず10パターン出させた
「ダッシュボードのデザイン案を10パターン、HTMLで作って」と指示。全部ブラウザで見比べて、一番良かったパターンを選んだ。

Step 2: 好みを具体的に伝えた
・テーマカラーはコーラル × スカイブルー
・絵文字は使わない（テキストかCSSのドットで代替）
・背景はCanvasアニメーション（ゆっくり動く光の球）
・カードの背景に銀色っぽいシマー（光沢感）
・サイドバーは白系

Step 3: カンバン方式を採用
タスクをドラッグ&ドロップでカラム間移動できるように。「未確認→確認済み→実行中→完了」のように、タスクの進行状況が一目で分かるレイアウトに。

ポイント: 最初から完璧を目指さなくてOK。「ここをもっと○○にして」「この部分の色を変えて」と少しずつ調整していくのが一番効率的です。
